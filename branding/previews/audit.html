Token Coverage Gaps

Where hardcoded values leak through
Fixed Black labels on accent buttons (dark mode) Sev · Critical
.ts-btn--primary, .ts-chip--active, and .ts-badge--accent hardcoded color:#0c0d0f. With the orange accent (oklch L≈0.65) this drops below WCAG 4.5:1 in dark mode. The source already shipped --ts-on-accent via OKLCH but it wasn't wired into our extract.
FIX · Added --ts-on-accent-threshold: 0.72 + computed --ts-on-accent; replaced 3 hardcodes. Pure-white labels on orange now.
Fixed Status buttons hardcoded near-black text Sev · High
.ts-btn--success used color:#06210f; .ts-btn--danger used color:#2a0606. Same OKLCH problem at scale: invisible on dark, brittle if status colors are retuned.
FIX · Added --ts-on-success, --ts-on-warning, --ts-on-danger, --ts-on-info using the same auto-contrast formula.
Drift Orange ramp uses literal hex, not derived Sev · Medium
--ts-orange-{300..800} are hex literals. If the user retones --ts-accent-h to swap brand color, the ramp doesn't follow.
FIX · toolskin-extras.css ships a derived --ts-accent-{50..900} via oklch(from var(--ts-accent) L C h). Single source of truth.
Drift Orange-tinted rgbas baked in CSS Sev · Medium
rgba(255, 85, 0, 0.06), 0.15, 0.25, 0.35 appear 14× as ghost / dim / border tints. Same retheming break.
FIX · Replace with color-mix(in srgb, var(--ts-accent) X%, transparent) in next pass; tokens already exist (--ts-orange-dim/ghost) but aren't applied site-wide.
Pass Spacing scale (4pt grid) Sev · —
--ts-sp-1…--ts-sp-12 applied consistently. No hardcoded margins in surveyed components.
Pass Radius scale, font tokens, surface tiers Sev · —
All radius / font-family / bg-N usage routes through tokens.
