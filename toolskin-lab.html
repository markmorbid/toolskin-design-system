Toolskin UI Kit Lab

Vanilla-only components in assets/js/toolskin-uikit.js; UI Kit styles are merged into assets/css/toolskin.css. No jQuery. Prefix ts-ui-* and API window.ToolskinUIKit. Reference widgets mirror the jQuery UI showcase patterns (accordion, spinner, sortable, etc.) without external UI libraries.

Checkbox & Radio (ts-* + ts-ui-*)

:root sets --ts-ui-control-size / --ts-ui-control-mark-inset so inputs stay sized even without a wrapper. Wrappers .ts-checkbox / .ts-radio add --md|lg|xl (md is the smallest documented size). Rows: .ts-ui-control-row or .ts-checkbox-group. Radio: .ts-radio-group = flush container, row padding via --ts-ui-radio-group-item-py / --ts-ui-radio-group-item-px; compact = .ts-radio-group--compact or .ts-radio-group.compact (single pane, dividers), outer inset --ts-ui-radio-group-compact-padding (default 0).

Masonry (core + UI Kit)

.ts-masonry uses a dense-packed grid (grid-auto-flow: dense) so multi-cell items do not leave large empty holes. Same modifiers as index.html. .ts-ui-masonry--grid is a uniform auto-fill tile grid. .ts-ui-masonry--lanes: with grid-lanes you get lane packing where supported; otherwise a flex-wrap layout with .ts-ui-masonry__item--photo, per-tile --ts-ui-photo-aspect, --ts-ui-masonry-item-flex-grow, and object-fit: cover.
